# Contracts: Health Endpoints

This feature exposes two HTTP endpoints. The authoritative machine-readable contract is
generated by Goa from the `design/` package into `gen/http/openapi*.yaml|json` at build time
(`make apigen`). The file [`health.openapi.yaml`](./health.openapi.yaml) in this folder is a
hand-authored excerpt that captures the intended contract for review before generation.

## Summary

| Method | Path      | Auth | Success                         | Failure                          | Docs |
|--------|-----------|------|---------------------------------|----------------------------------|------|
| GET    | `/livez`  | none | `200` `text/plain` body `OK\n`  | (none in-band; no response ⇒ dead)| hidden (`swagger:generate=false`) |
| GET    | `/readyz` | none | `200` `text/plain` body `OK\n`  | `503 Service Unavailable`        | hidden (`swagger:generate=false`) |

## Notes

- **Unauthenticated**: neither endpoint declares a Goa `Security(...)` scheme; they must not be
  gated by the JWT/Heimdall middleware. Kubernetes probes call them directly on the pod's `web`
  port, not through the API gateway.
- **Body contract**: exactly the two characters `OK` followed by a single newline (`OK\n`).
- **Excluded from public API docs**: both methods set `Meta("swagger:generate", "false")`.
- **Probe mapping** (existing chart, unchanged): `livenessProbe → /livez`, `readinessProbe → /readyz`,
  `startupProbe → /readyz`.
