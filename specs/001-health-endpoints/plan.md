# Implementation Plan: Health Endpoints (Readyz & Livez)

**Branch**: `feat/add-health-endpoints` | **Date**: 2026-07-01 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/001-health-endpoints/spec.md`

## Summary

Bring the campaign service from a non-compiling Goa scaffold to a compiling, runnable HTTP service that exposes two unauthenticated operational endpoints — `GET /livez` (liveness) and `GET /readyz` (readiness) — each returning `text/plain` with body `OK\n`, and `/readyz` returning `503` when not ready. The approach mirrors the reference `lfx-v2-project-service`: define the endpoints in a Goa design package, generate the HTTP layer, implement a service type satisfying the generated interface, and wire it through the dependency container. The build system is completed alongside: add `fmt`/format-check and a production-build target to the `Makefile`, and repair the scaffolded (currently non-functional) branch/main/tag container-publish workflows to full parity with the project service (Cosign signing, SLSA provenance, SBOM, multi-arch).

## Technical Context

**Language/Version**: Go 1.24.2

**Primary Dependencies**: Goa v3 (`goa.design/goa/v3` v3.25.3) for API design + code generation; `goa.design/clue` (debug/log); `go.opentelemetry.io/contrib/instrumentation/net/http/otelhttp`; `log/slog` structured logging; `stretchr/testify` for tests. (The `nats.go` dependency is present in the module but is NOT used by this feature.)

**Storage**: N/A (no persistence; no external runtime dependencies wired in this feature)

**Testing**: `go test -race` with table-driven unit tests (testify), matching the reference `service_endpoint_test.go` pattern for `Readyz`/`Livez`.

**Target Platform**: Linux containers on Kubernetes; published images target `linux/amd64` and `linux/arm64`.

**Project Type**: Single Go HTTP web service (Goa-generated transport + hand-written service implementation).

**Performance Goals**: Liveness response < 100 ms (SC-002); probe handlers must be cheap (no downstream calls).

**Constraints**: Both endpoints unauthenticated (no JWT/Heimdall); body is the literal `OK\n` with `text/plain`; endpoints excluded from generated public API docs (`Meta("swagger:generate", "false")`); readiness returns `503` before initialization completes.

**Scale/Scope**: Two endpoints + one service implementation + container wiring + Makefile targets + three CI workflows. Small, self-contained change.
